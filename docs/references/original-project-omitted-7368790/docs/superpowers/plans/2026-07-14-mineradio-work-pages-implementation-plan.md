# Mineradio 工作页统一实施计划

状态：本阶段已完成
日期：2026-07-14
设计依据：`docs/superpowers/specs/2026-07-10-work-pages-ui-polish-design.md`

## 目标

在不修改影院大厅、媒体队列和顶部导航的前提下，把总览、控制室、任务中心、日历、发现和设置统一为基于 Mineradio 源码的深色雾面工作页，并保证所有链路状态都由真实证据驱动。

## 已完成

1. 总览：建立“订阅 → 获取 / 下载 → 进入 115 → 入库”四步链路摘要。
2. 控制室：固定 Torra、qBittorrent、Symedia、Emby 四个核心服务和只读检查台。
3. 任务中心：先接入真实 qB 下载证据，随后完成统一任务链，关联本地订阅、Torra、qB、115 邻接证据、Symedia 和 Emby。
4. 日历：统一月份控制、待播出、已入库和逾期未入库状态；逾期条目可进入任务中心。
5. 发现：形成内容发现与订阅中枢双区工作台；订阅默认显示 PT 通道与八分类结果。
6. 设置：明确 PT / Torra 始终优先、自动云盘兜底默认关闭、环境变量和认证凭据只读。
7. 视觉：工作页取消金色和亮青主视觉，使用深黑、银蓝、深电蓝、暖白和玫红；保留影院大厅与顶部导航原样。

## 真实环境核对

- 2026-07-14 已只读连接局域网 qBittorrent。
- qBittorrent 版本：`v4.3.9`。
- 读取到 49 个任务，验证时有 1 个活跃下载。
- 中控 `/api/qbittorrent/summary` 已能读取同一实例的任务数、速度与状态。
- 登录凭据不写入仓库、页面或本文档。

## 验收结果

1. `npm test`：14/14 通过。
2. `npm run build`：通过。
3. 1440×900、1024×768、390×844：六个工作页无横向页面溢出。
4. 任务中心默认聚焦进行中任务；支持“全部 / 进行中 / 等待中 / 卡住 / 已入库 / 未关联”六种筛选，每种首次展示 12 条并可继续加载。
5. 日历在手机尺寸保持 860px 最小网格，通过面板内部横向滚动查看。
6. 发现页订阅中枢使用独立滚动；手机海报双列，避免页面高度失控。
7. 设置页连接信息在手机降为单列，环境变量名称保持只读。
8. 浏览器控制台无错误或警告。
9. 影院大厅、媒体队列和顶部导航未修改。

## 后续阶段

1. fnOS 写链路条件满足后，再进行 Torra 真实单条推送验证；下载器 ID 已核对，但真实推送继续关闭。
2. 云盘策略后端接入后，再把设置页的“自动云盘兜底”升级为可保存开关；默认仍为关闭，PT 始终优先。
3. 资源来源和转存链稳定后，在发现页增加资源搜索、候选预览和人工转存工作区。

## 安全边界

- `TORRA_PUSH_ENABLED=false` 保持默认关闭。
- 不用 qB 下载完成推断“已进入 115”或“已入库”。
- 不在浏览器保存服务账号、密码或 Token。
- 不在控制室提供删除、清空和批量文件操作。
- 不修改影院大厅、Mineradio iframe、媒体队列和顶部导航 UI。
