# PT 核心版阶段 0-1 实施清单

状态：实施中

依据：`2026-07-14-dual-channel-subscription-center-design.md`

## 本批目标

本批只建立 PT 主链的安全底座，不迁移 SQLite、不改工作页 UI、不接入云盘能力，也不触发现网 Torra 写操作。

1. 冻结本地 138 条订阅、订阅配置和活动记录。
2. 建立统一媒体身份与 8 分类解析规则。
3. 固化 Torra/qB 分类保存路径模板。
4. 在创建 Torra 订阅前查询已有订阅，按 TMDB、类型和季号查重。
5. 增加 dry-run 预览，展示分类、保存路径、下载器和阻塞原因。
6. 真实 Torra 推送增加显式开关，默认关闭。
7. 无法可靠分类时允许订阅级人工选择八分类，并提供只读批量审计报告。

## 安全边界

- `TORRA_PUSH_ENABLED` 默认关闭。
- `save_path` 为空时禁止推送。
- 下载器 ID 未核实时禁止推送。
- Torra 订阅列表查重失败时禁止创建新订阅。
- 已有 Torra 订阅只复用其 ID，不覆盖其自定义路径和规则。
- 影院大厅、Mineradio、媒体队列和顶部导航不在本批改动范围。

## 验证标准

- 8 个分类均生成 `/vol02/1000-4-32d3f6a0/torra/<分类>`。
- 日漫和国漫写入 `is_anime=true`。
- 电影、剧集、动漫和综艺分类测试通过。
- 同 TMDB、类型和季号的 Torra 订阅不会再次调用保存接口。
- 查重接口失败时不会调用 Torra 保存接口。
- 原有核心稳定性测试和 TypeScript 构建通过。
