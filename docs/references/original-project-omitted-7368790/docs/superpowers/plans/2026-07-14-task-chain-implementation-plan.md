# 统一任务链实施计划

状态：本阶段已完成
日期：2026-07-14
设计依据：`docs/PLAN.md` §6“任务链路服务”、`docs/superpowers/specs/2026-07-14-dual-channel-subscription-center-design.md`

## 目标

在媒体控制中心内建立一条可核验的媒体任务链，把本地订阅、Torra、qBittorrent、115、Symedia 和 Emby 的真实证据关联到同一媒体单元。PT / Torra 是默认主通道；云盘只作为受开关控制的后续兜底能力，本阶段不启用自动云盘动作。

## 实施步骤

1. 扩展 Torra、Symedia 和 Emby 只读适配器，读取任务关联需要的 TMDB、季集、文件路径、状态和索引字段。
2. 建立 `taskChainService`，以本地订阅为主干，并行拉取外部服务证据。
3. 按“TMDB 强关联 → Torra 文件名桥 → 标题与季集兜底”匹配任务；置信度不足时保留为未关联行。
4. 将状态统一为“订阅 → 获取 / 下载 → 进入 115 → 入库”四步；115 无直接 API 的部分明确标记推断。
5. 暴露 `GET /api/tasks/chain`，返回任务行、计数和服务连接摘要。
6. 任务中心接入统一接口，提供“全部 / 进行中 / 等待中 / 卡住 / 已入库 / 未关联”筛选，每次渐进显示 12 条。
7. 增加任务链匹配和状态推导回归测试，并完成桌面、平板和手机视口验收。

## 已完成结果

- 本地订阅成为任务主干；无本地订阅的 qB / Symedia 记录不会丢失，而是显示为未关联外部任务。
- Torra 通过 TMDB ID、媒体类型和季号关联订阅；qB 通过 Torra 下载文件名或下载分集文件桥接。
- Symedia 使用 tmdbid、season、episode 和 src 强关联；Emby TMDB 索引只负责最终入库确认。
- 下载完成且没有 Symedia 记录时，115 步骤显示“推断处理中”；超过 6 小时显示“推断卡住”。
- 卡片展示证据来源、当前步骤、进度、卡住原因和打开原工具建议，不提供破坏性操作。

## 真实环境验证

- qBittorrent：v4.3.9，49 个任务。
- Torra：首轮字段快照 211 条；210 条有 TMDB ID，87 条有下载文件名，50 条有下载分集记录，211 条有保存路径；最终接口复验实时总数为 210，下载器 ID 已只读核对。
- Symedia：53,121 条历史；最近 200 条样本均有 TMDB ID、源路径和目标路径，197 条有季集字段。
- `/api/tasks/chain` 最终快照：235 行；进行中 36、卡住 56、已入库 45、等待中 98、未关联 97。进行中与等待中属于实时状态，会自然变化。
- qB、Torra、Symedia 在线；本次临时验证未配置 Emby，因此 Emby 结果保持“尚未确认”。

## 验收标准

- `npm test` 14/14 通过。
- `npm run build` 通过。
- 任务中心在 1440×900、1024×768、390×844 无横向页面溢出，手机四步链为 2×2。
- 浏览器控制台无错误或警告。
- 影院大厅、Mineradio iframe、媒体队列和顶部导航未修改。
- 仓库和文档中没有真实账号、密码或 Token。

## 安全边界与后续

- `TORRA_PUSH_ENABLED=false` 继续保持关闭；本阶段不做真实推送、搜索、保存、转存或删除。
- PT 始终优先。自动云盘兜底只有在全局开关或订阅级开关明确启用、PT 达到兜底条件且重复状态核对通过后才允许进入。
- 后续再接可回滚控制动作、真实 Emby 索引验证、云盘 Resource Gateway 和审计状态机。
