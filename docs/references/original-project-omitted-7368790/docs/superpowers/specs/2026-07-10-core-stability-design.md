# 媒体控制中心核心稳定性修复设计

实施状态：已完成（2026-07-10）。

## 背景

当前版本已经完成 Emby、qBittorrent、Torra、Symedia、订阅中枢、发现页和日历页的主体接入，但检查发现 Docker 数据持久化、自动订阅快照、外部标题识别和部分统计逻辑存在确定缺陷。这些问题会影响现有数据可靠性，应在任务链路开发前处理。

应用自身的访问密钥保护属于 VPS 上线前的必要能力，但本轮按用户决定只记录计划，不进入实现，避免扩大当前稳定性修复范围。

## 本轮范围

1. Docker 将 `/app/data` 挂载到持久卷，容器重建后保留订阅、配置和活动日志。
2. Docker Compose 补齐 Emby 账号密码与 Symedia Token/账号密码环境变量。
3. 自动订阅默认关闭，README 同步为每日定点调度的真实行为。
4. 自动订阅只有在所有已选择来源均成功读取时才提交完整快照；来源失败时保留原有自动订阅，返回错误并记录活动日志。
5. 单个条目 TMDB 识别失败仍按现有过滤规则跳过，但不会因为来源级故障清空旧快照。
6. TMDB 搜索前剥离标题末尾的季数标记，例如“第五季”“Season 5”“S05”，并把提取到的季号写入订阅条目。
7. 总览“未来 3 天”跨月时同时读取下个月日历。
8. Symedia 今日入库统计按分页读取当天所需记录，不再最多只统计最近 50 条；最近失败仍明确限定为取样窗口。
9. 增加最小 Node 测试，覆盖标题季数解析、自动快照提交条件和跨月日期范围。

## 不在本轮范围

- 不实现应用登录页、访问密钥、会话 Cookie 或权限模型。
- 不实现任务链路聚合接口。
- 不增加 qB、Emby 等写操作。
- 不升级整套前端框架或进行无关重构。

## 设计

### Docker 数据与配置

Compose 使用命名卷挂载 `/app/data`。运行数据继续由现有 JSON 存储管理，不改变文件格式。认证环境变量只增加传递，不在页面或日志中输出明文。

### 自动订阅提交边界

每轮运行分别记录成功来源和失败来源。只要存在来源级读取失败，本轮不调用 `applyAutoSnapshot`，已有自动订阅不会被不完整结果覆盖。系统仍记录本次尝试时间，避免每日定时器在计划时间后每分钟重复请求失败来源；用户可以通过手动刷新立即重试。

条目级过滤和 TMDB 未匹配不视为来源级失败，因为这是正常的数据质量结果；这些条目继续记录为跳过。

### 标题与季号识别

增加一个纯函数，将来源标题拆为基础标题和可选季号。仅处理标题尾部明确的季数格式，避免误删片名正文。TMDB 使用基础标题搜索；剧集订阅保存提取的 `seasonNumber`，没有明确季号时维持现有行为。

### 日期与统计

总览根据今天和结束日期涉及的月份请求一到两个月份，再按日期范围统一计数。

Symedia 从第一页开始读取历史记录，直到记录日期早于今天、无更多数据或达到安全页数上限。累计记录仍使用接口返回的 `total`，最近失败只统计固定取样窗口并在界面文案中说明。

## 错误处理

- Docker 卷不存在时由 Compose 自动创建。
- 来源级失败不会修改订阅快照，但会记录尝试时间和错误，防止定时器形成重试风暴。
- TMDB 搜索失败按单条跳过处理，不中断其他条目。
- Symedia 翻页中途失败时返回连接失败，不展示不完整的“今日入库”真值。

## 验收标准

- `docker compose config` 能看到数据卷和完整认证变量映射。
- 新配置默认不会启动自动订阅定时任务。
- 任一订阅来源失败时，原自动订阅数量和内容保持不变。
- “疑犯追踪 第五季”使用“疑犯追踪”搜索并保存季号 5。
- 月末总览能统计下月前三天内的播出条目。
- Symedia 当日超过 50 条记录时今日计数仍准确。
- 自动测试与 `npm run build` 通过。

## 后续计划：VPS 登录保护

VPS 上线前增加 `MCC_ACCESS_KEY`。推荐采用应用登录页、恒定时间密钥比较和 HttpOnly 签名会话 Cookie，保护全部 `/api` 接口并限制 CORS。生产环境未配置密钥时应拒绝启动。该能力单独设计、实现和验证，不与本轮核心稳定性修复混合。
