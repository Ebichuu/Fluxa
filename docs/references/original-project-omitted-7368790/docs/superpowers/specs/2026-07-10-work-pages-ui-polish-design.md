# 工作页统一 UI 与双通道框架改造设计

状态：已完成六个工作页改造与多尺寸验收

## 2026-07-14 重设计方向

现有六个工作页只视为功能草稿，不保留其卡片排列和页面视觉。影院大厅保持不变，顶部胶囊导航继续冻结；工作页以 `D:\Mineradio\resources\app\public\index.html` 的真实源码为主要视觉依据，形成同一产品的控制层：

- 深黑 `#08090b`、黑纸 `#0e1014` 与白色文字构成基础。
- 半透明雾面玻璃、20–24px 大圆角、白色内高光、细灰边框和低对比阴影。
- 胶囊状态控件、圆形工具按钮和封面式层级延续 Mineradio 语言。
- 取消工作页原计划中的金色和青色主视觉，不使用香槟金或荧光青作为状态骨架。
- PT 活跃状态使用深电蓝 `#2442ff`，普通连接与次级强调使用银蓝 `#9db8cf`，完成/入库使用暖白 `#f8f4ee`，异常使用玫红 `#ff5367`。
- 标题使用 Mineradio 的 `Noto Sans SC / Inter` 高字重体系，路径、速度、数量和时间使用 `JetBrains Mono` 等宽数据字体。
- 背景和卡片只使用弱色晕与细线纹理，不复制 Mineradio 的唱片、歌词、骷髅点云和 3D 舞台对象。
- 不复制影院大厅的 3D 海报、播放互动和媒体队列布局。

工作页的识别性元素是“媒体链路轨道”：用真实的四步证据表达 `订阅 → 获取/下载 → 进入 115 → 入库`。没有证据的步骤必须显示“待接入证据”，不能用视觉效果伪装成已完成。

首个样板先重做总览和控制室：

- 总览验证链路轨道、运行摘要、正在获取、最近入库和最近订阅的层级。
- 控制室验证 Torra、qB、Symedia、Emby 四个核心服务与右侧检查台的交互。
- 2026-07-14 样板已确认，同一壳层和状态语言已铺到任务中心、日历、发现和设置。
- 任务中心已接入统一任务链；订阅、Torra、qB、Symedia 和 Emby 使用真实证据，115 无直接 API 的部分明确标记“推断”，没有可靠关联时显示“未关联”。

## 背景

总览、控制室、任务中心、日历、发现和设置已经具备基础页面结构。新的信息架构以 PT/Torra 订阅主链为核心，云盘仅作为人工补资源和可选兜底能力。本轮同时调整六个工作页的页面职责、数据表达和视觉层级。

影院大厅首页、Mineradio iframe、媒体队列面板及其交互不在本轮范围。共享顶部导航以用户提供的当前胶囊导航为冻结基线，保持结构、位置、尺寸、图标、字号、间距、颜色、激活样式和响应式行为不变。页面样式必须使用工作页作用域，避免影响首页和导航。

顶级导航继续保持：总览、影院大厅、控制室、任务中心、日历、发现，以及现有设置按钮。不新增“订阅中心”顶级入口；订阅中心并入发现页内部。

## 目标

1. 六个工作页形成同一套深色控制台视觉语言。
2. 页面结构真实反映统一订阅台账、PT 主链和下游入库链路；云盘仅在启用或实际参与时出现。
3. 用户进入页面后能快速识别页面标题、核心状态、当前通道、主要内容和可执行操作。
4. 正常、完成、警告、失败和“证据不足/推断”状态有明确区分。
5. 桌面端提高信息层级与空间利用率，移动端避免挤压、逐字断行和不可操作的小控件。
6. 不改变影院大厅和顶部导航 UI。

## 视觉系统

### 工作页背景

工作页使用独立的深色渐变基底与低对比环境光，不修改影院大厅背景。环境光只负责区分页面层级，不使用大面积霓虹或持续动画。

### 面板层级

- 普通 `panel`：信息与状态层，边框轻、背景较透明。
- `console-panel`：操作与控制层，背景更深、边框和内阴影更明确。
- 交互卡片：悬停时轻微上移、边框变亮；键盘焦点必须与悬停同等清晰。
- 异常状态：只在需要注意的边框、状态点和关键数字上使用玫红，不整块染色。

### 排版与间距

- 页面标题区域缩短纵向高度，标题保持轻字重和明确行宽。
- 页面主体统一最大宽度、左右安全间距和区块间距。
- 标签、辅助文字和时间信息降低对比度，关键数字与当前状态提高对比度。
- 按钮、分段标签、输入框统一高度、圆角、边框和焦点样式。

### 动效

只使用 120–220ms 的透明度、位移和边框过渡。支持 `prefers-reduced-motion`，关闭非必要位移和缩放。

## 页面设计

### 总览

- 顶部使用轻量运行状态带，优先展示 Torra、qB、秒传、入库、活动任务、失败和待人工处理数量。
- 主内容按“正在获取 / 最近入库 / 最近订阅与兜底”组织，不再只围绕 qB 下载。
- 当前通道使用明确标签：`PT`、`PT → 云盘兜底`、`人工云盘`、`仅跟踪`；默认绝大多数任务应显示为 `PT`。
- 异常项优先显示原因、等待时长和建议入口；不增加装饰性监控图表。

### 控制室

- 核心服务卡优先展示 Torra、qB、Symedia 和 Emby；Resource Gateway、115 和 Refind 归入辅助服务分区。
- 服务卡网格与右侧控制面板形成清晰主从关系。
- 服务卡突出一个核心指标，健康状态、检查时间、队列或失败信息依次降级。
- 选中服务卡使用轮廓、背景和轻微位移表达，不依赖大面积高亮。
- 右侧分段标签按“状态 / 最近任务 / 安全操作”组织。
- 只提供连接测试、刷新、打开原工具、人工重试等可审计操作；不提供 115 批量删除和清空操作。

### 任务中心

- 数据源改为统一任务链，不再把 qB 任务直接伪装成完整媒体链。
- 统一任务链筛选使用“全部 / 进行中 / 等待中 / 卡住 / 已入库 / 未关联”，并显示各状态计数；“进行中”只包含真实 active，不混入 waiting。
- 每个筛选首次展示 12 条，按需每次继续展示 12 条，避免大量历史任务首次进入时铺满页面。
- 有 qB 关联的任务卡提供暂停/恢复入口；一次作用于当前媒体任务关联的全部下载，必须经过二次确认、服务端状态复查和活动日志审计。
- 控制室 Emby 操作区使用 Symedia 最新成功入库与 Emby 最新索引时间作为刷新证据；无新证据时禁用，全库刷新手动确认并设 10 分钟冷却。
- 任务卡的标题、订阅策略、当前通道、状态、四步链和操作区形成稳定层级。
- 四步链固定为“订阅 → 获取/下载 → 进入 115 → 入库”，每一步标注已验证、推断或失败证据。
- 卡住任务突出卡住时长、原因和建议动作；普通任务减少视觉噪声。
- 展开区展示 PT 尝试、云盘候选、规则匹配、兜底切换、115 转存和 Symedia 结果。

### 日历

- 月份切换、统计和图例形成统一页头。
- 今天使用深电蓝边界，已入库使用暖白标记，逾期未入库使用玫红边界；不使用金色和亮青作为工作页状态色。
- 条目增加策略与获取状态提示，但不把单元格变成完整任务卡。
- 点击条目打开订阅详情，可查看当前通道、是否允许云盘兜底和对应任务链。
- 桌面保持七列网格；小屏允许日历主体横向滚动并保持最小单元格宽度，避免内容压缩失真。

### 发现

- 当前发现页使用“内容发现 + 订阅中枢”双区工作台，不增加顶级导航；资源搜索在真实候选源和转存链路接入后再增加内部工作区。
- 内容发现保留全球日播、TMDB、海外流媒体、豆瓣和平台热更来源。
- 我的订阅以 PT 状态、分类、外部绑定和库内进度为主；云盘兜底开关作为次级设置，不占用主卡片核心位置。
- 资源搜索按需打开，展示 Telegram/HDHive 候选、格式标签、规则结果、是否需要积分和人工转存操作。
- 来源和内部工作区使用紧凑横向入口，选中状态清晰但不过度发光。
- 搜索、当前筛选摘要和重置操作放在同一工具区。
- 筛选项按行组织，降低标签噪声。
- 海报卡强化图片、标题和订阅按钮，简介维持辅助层级。
- 创建订阅时必须明确媒体分类，策略默认固定为 PT；无法分类时进入人工确认，不默认进入未分类目录。

### 设置

- 当前设置按“影院大厅视觉、获取通道、服务连接、订阅扫描与来源”形成清晰分组；通知与应用登录安全在对应后端能力接入时单独增加。
- 设置页以 PT/Torra 默认规则为主；“自动云盘兜底”放在高级辅助功能中，默认关闭。关闭自动兜底不影响人工资源搜索和确认转存。
- 默认策略为 PT；开启云盘后可设置订阅级继承、开启或关闭兜底。
- PT 切换到云盘的等待条件和失败条件必须使用解释性文案，不能只显示一个模糊数字。
- 配置状态、说明、输入/选择控件和操作按钮对齐统一网格。
- 环境变量和认证凭据继续只读展示；页面只允许保存非敏感策略和开关。

## 响应式规则

- 1200px 以上使用双栏或多栏控制台布局。
- 760–1199px 缩减侧栏宽度和网格列数，保留主要信息层级。
- 760px 以下切为单栏；导航继续横向滚动；控制面板移到内容下方；日历使用可滚动最小宽度。
- 所有按钮最小可操作尺寸为 40px，标题和长路径允许正常换行或省略，不逐字断行。
- 顶部导航在所有断点下继续使用现有响应式规则，不因工作页新增内容改变高度和排布。

## 实现边界

- 优先修改共享 CSS 和现有 class，不复制页面级内联样式。
- 只有现有结构无法表达层级时才增加少量语义容器或状态摘要。
- 工作页允许随双通道框架调整 API、数据类型和内部组件结构。
- 不新增顶级页面路由；订阅中心和资源搜索并入发现页。
- 不修改 `AppTopNav`、`MediaHall`、`MineradioEmbed`、`MediaQueuePanel` 和 Mineradio 静态资源的视觉结构。
- 全局样式修改必须验证不会命中影院大厅和顶部导航选择器。
- 不引入新的 UI 框架、字体包、图片、视频或音频资源。
