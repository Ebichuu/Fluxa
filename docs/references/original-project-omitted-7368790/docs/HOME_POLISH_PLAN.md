# 首页完善计划

更新时间：2026-07-07

## 当前方向

首页已经切换为 Mineradio 原始视觉 iframe 嵌入路线。后续完善优先围绕“媒体中心首页体验”展开，不再继续旧的逐模块视觉重写方案。

## 已完成

- `/mineradio/embed` 动态注入桥接脚本和嵌入样式。
- React 首页通过 `postMessage` 把媒体库、媒体项、当前焦点和视觉参数传入 iframe。
- iframe 内覆盖播放入口，只保留视觉和 shelf 交互。
- 右侧 Mineradio shelf 可展示媒体库和媒体项。
- 左侧新增媒体库 / 当前队列面板，可切库、切当前媒体项并常开固定。
- iframe 加载态、启动慢提示和失败提示已接入首页。
- 旧的 React/TypeScript 视觉重写源码和样式已删除。

## 继续优化

1. 浏览器级回归：桌面和移动视口检查 iframe 非空、左侧面板不遮挡导航、底部状态不溢出。
2. 左侧面板细节：当前项自动滚入视野、长标题压缩、空库状态提示。
3. iframe 桥接稳定性：继续压测切库、快速滚轮、方向键和 shelf 点击同步。
4. 文案边界：所有提示都使用媒体库语义，避免出现音乐平台、播放器、歌词、播客等业务措辞。
5. 生产部署：Mineradio public 已内置到 `vendor/mineradio-public`，确认 Docker 镜像默认可直接显示首页视觉。

## 不做

- 不恢复 Mineradio 播放器底栏。
- 不迁音乐平台歌单、登录、搜索、歌词、播客、评论和收藏。
- 不再新增独立 Three.js 视觉重写模块。

## 验证

- `npm run build`
- 旧路线符号扫描
- `/`、`/mineradio/embed`、`/api/media/home` 端点检查
- 注入脚本语法解析
- 浏览器检查左侧面板、右侧 shelf、当前焦点同步
