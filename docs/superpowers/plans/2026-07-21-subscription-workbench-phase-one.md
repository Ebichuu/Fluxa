# 第一阶段订阅工作台实施计划

设计依据：`docs/superpowers/specs/2026-07-21-subscription-workbench-phase-one-design.md`

## 任务 1：保护订阅台账

- 将自动来源刷新从整体覆盖改为按稳定 key 增量合并。
- 保留手动订阅、Torra 镜像和不属于本轮来源的既有订阅。
- 增加回归测试，覆盖手动条目和 Torra 镜像不会丢失。

## 任务 2：订阅工作台聚合接口

- 新增只读工作台快照服务与 `/api/v2/subscriptions/workbench`。
- 聚合五项能力、四项统计、最后读取时间和条目级链路摘要。
- 外部能力采用部分成功模型，返回明确状态和阻塞原因。

## 任务 3：前端工作台接线

- 增加工作台类型和 API 客户端。
- 顶部接入五项真实能力状态、统计、刷新和重试。
- 空台账显示 Torra 预览与确认导入引导。
- 列表直接显示范围、缺集、Torra、qB、115、入库和阻塞原因。
- 写入关闭时禁用写操作。
- 将“执行一轮”改为“更新自动订阅来源”。

## 任务 4：验证

- 运行订阅、Torra 同步和任务链相关 Python 测试。
- 运行完整 Python 测试、TypeScript 类型检查和生产构建。
- 运行 `git diff --check`，确认没有覆盖用户现有改动。
- 不在测试中访问生产 Torra、qB、RSS 或下载接口。
