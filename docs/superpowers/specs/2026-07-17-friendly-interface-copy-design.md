# 媒体控制中心友好文案设计

日期：2026-07-17

## 目标

媒体控制中心既要让维护者看见真实工具和证据，也要让家人或偶尔使用者不理解 Torra、qB、Symedia 等术语时仍能完成找片、订阅和查看进度。

本轮只调整展示文案与文字选择行为，不改 API、后台状态、订阅规则、推送逻辑或 Mineradio 核心视觉。

## 信息分层

第一层使用中文业务语：正在下载、等待入库、需要处理、已经完成。页面标题直接说明这一页能做什么。

第二层保留技术依据：PT、Torra、qB、115、Symedia、Emby、TMDB、匹配方式和原工具入口。它们出现在服务卡、任务详情、预检信息或次级文字中，不与第一层标题争夺注意力。

## 页面文案

| 页面 | 中文短标 | 主标题 |
| --- | --- | --- |
| 总览 | 总览 · PT 主链 | 从订阅到入库，一眼看清进度。 |
| 发现 | 发现 · 找片 | 找到想看的内容，加入订阅即可。 |
| 我的订阅 | 订阅 · 自动获取 | 管理正在追的电影和剧集。 |
| 任务中心 | 任务中心 · 处理进度 | 媒体任务，现在进行到哪一步。 |
| 日历 | 日历 · 播出与入库 | 什么时候播，哪些还没入库。 |
| 控制室 | 控制室 · 服务状态 | 查看各项服务是否正常。 |
| 订阅设置 | 订阅设置 · 来源与时间 | 设置系统自动发现哪些内容。 |
| 系统设置 | 设置 · 连接与安全 | 管理服务连接与访问保护。 |

## 状态词

- `strong`：已精确匹配。
- `fallback`：按标题推测。
- `unlinked`：尚未接到链路。
- `Torra / qB` 表层显示“正在下载”。
- `Symedia / Emby` 表层显示“整理与入库”。

后台字段和接口响应保持原样，前端负责转换成用户能读懂的文字。

## 影院大厅与复制

影院大厅外露状态统一为“Emby 实时媒体库”“示例媒体库”“正在连接媒体库”。Mineradio 内部视觉预设名和控制台术语不修改。

页面正文、任务名、错误信息、路径和代码允许复制；导航、按钮和其他交互控件继续禁止选择，避免误操作时出现选区。
