# Fluxa 证据所有权、集级日历与 RSS 身份设计

日期：2026-07-22
状态：已确认
范围：任务串链、健康分类、首页摘要、日历接口与移动端、RSS 身份和导航收口

## 1. 背景

当前实机数据暴露出四类相互关联的问题：

1. 空 TMDB ID 被当成精确相等，导致多条无关电影订阅继承同一条 Symedia 文件错误。
2. 低分或不覆盖等保护性拒绝仍可能进入红色异常。
3. 季级任务证据被重复应用到同季多集，造成日历批量显示已获取或已入库。
4. RSS 索引只保存标题、简介、季集和版本信息，没有结构化保存 TMDB/IMDb 身份，也没有中文详情视图。

本次修复以证据正确性为先。前端不得通过隐藏、去重或改文案掩盖错误关联。

## 2. 目标

- 不相关媒体不能共享同一条 qB 或 Symedia 证据。
- 串链结果与输入遍历顺序无关。
- 正常保护、真实失败、未知证据和成功使用稳定、可解释的优先级。
- 日历只使用明确的集级证据，不从季级完成推导单集完成。
- 移动端可以直接进入日历，并使用无横向滚动的紧凑视图。
- 首页只显示用户可理解的错误摘要，技术证据继续保留在任务详情。
- RSS 在不访问 PT 详情页的前提下保守提取媒体身份，并记录身份来源和置信度。
- 旧客户端和旧请求保持兼容。

## 3. 非目标

- 不自动抓取 PT 站点种子详情页。
- 不把季级任务强制拆成新的顶层集级任务链。
- 不修改 Torra、qBittorrent、Symedia 或 Emby 的业务职责。
- 不根据模糊标题自动认领媒体身份。
- 不删除旧日历或 RSS 接口字段。
- 不在本阶段补齐历史任务事件时间线。

## 4. 证据候选与统一裁决

### 4.1 两阶段流程

串链必须拆成两个阶段：

1. 候选收集：为每条外部证据收集所有可能目标，不产生所有权。
2. 统一裁决：在完整候选集合上按规则分配唯一所有者。

禁止在遍历订阅或任务时使用“找到一个就绑定”的实现。输入顺序改变后，裁决结果必须保持一致。

### 4.2 证据记录

每条 qB、Torra 或 Symedia 证据至少记录：

- 'artifactKey'：证据或制品稳定身份；
- 'ownerTargetKey'：唯一目标，无法裁决时为空；
- 'matchMethod'：'tmdb_exact'、'title_season_unique'、'title_year_unique'、'artifact_exact' 或 'unresolved'；
- 'confidence'：'strong'、'fallback'、'conflict' 或 'unlinked'；
- 'conflictCandidates'：冲突目标列表；
- 'observedAt'：证据时间；
- 'source'：Torra、qBittorrent 或 Symedia。

同一证据可以服务同一目标的多个阶段，但不得归属于多个不同 'targetKey'。

### 4.3 候选规则

| 证据情况 | 裁决 |
| --- | --- |
| 有效 TMDB 且唯一对应 | 精确绑定 |
| 空 TMDB | 不参与精确匹配 |
| 电视剧无 TMDB | 类型、保守标题和季数全部一致，且候选唯一 |
| 电影无 TMDB | 类型、保守标题和年份全部一致，且候选唯一 |
| 多个候选都满足 | 不绑定，标记证据冲突 |
| 没有可靠候选 | 不绑定，标记证据不足 |

保守标题匹配使用规范化后的完整标题或已验证别名，不使用任意包含关系。电影缺少年份时不得使用标题回退。

### 4.4 冲突处理

- 多候选证据不分配给任何一个候选目标。
- 冲突证据作为独立未关联记录保留，健康状态为“证据不足”。
- 'reasonCode' 使用 'EVIDENCE_OWNER_CONFLICT'。
- 技术详情展示候选 'targetKey'，首页不展示内部 Key。
- 新旧串链只读对比必须统计新增绑定、解除绑定、冲突和未识别数量。

## 5. 健康状态与正常保护

### 5.1 任务级优先级

同一目标的最终健康状态按以下顺序裁决：

1. 存在任何真实失败：'action_required'；
2. 没有真实失败，但存在保护性拒绝：'protected'；
3. 所有必要阶段成功：'normal'；
4. 只有未知或不完整证据：'evidence_insufficient'；
5. 已确认仍在执行或等待计划：'waiting'。

'waiting' 只用于有明确运行或计划证据的情况，不能替代未知状态。

### 5.2 结构化保护规则

新写入证据优先使用结构化 'reasonCode'：

- 'QUALITY_SCORE_LOWER'：评分低于目标；
- 'QUALITY_WEIGHT_NOT_HIGHER'：权重低于或等于目标；
- 'QUALITY_OVERWRITE_CANCELLED'：取消覆盖；
- 'QUALITY_OVERWRITE_SKIPPED'：不执行覆盖；
- 'QUALITY_HIGHER_VERSION_EXISTS'：已有更高质量版本；
- 'DUPLICATE_RESOURCE_SKIPPED'：重复资源跳过。

文本识别只作为旧日志兼容层。兼容匹配结果同时记录 'matchedProtectionRule'，便于排查误判。

### 5.3 混合结果

- 同一目标只要存在一条真实失败，就不能被保护状态覆盖。
- 全部失败记录均属于保护规则时，显示蓝色“正常保护”。
- 保护项不提供重试按钮。
- 残留源文件可以在技术详情中提示，但不计入红色异常。

## 6. 集级证据

### 6.1 数据结构

任务聚合结果内部增加 'episodeEvidence'，每条至少包含：

- 'seasonNumber'；
- 'episodeStart'；
- 'episodeEnd'；
- 'numberingScheme'：'season_episode'、'absolute' 或 'special'；
- 'stage'：获取、下载、115、Symedia、STRM 或 Emby；
- 'artifactKey'；
- 'source'；
- 'observedAt'；
- 'matchMethod'；
- 'status'；
- 'reasonCode' 和 'reasonText'。

### 6.2 支持范围

必须覆盖：

- 'S01E03'；
- 'S01E03-E05'；
- 一个文件对应多集；
- 'S00' 特别篇；
- 动漫绝对集数；
- 季包中能够从文件列表明确解析出的集号。

绝对集数只有在现有媒体元数据能够可靠换算时才映射到季集；无法换算时保留为绝对集证据，不标记具体日历集。

### 6.3 日历使用规则

- 电影继续使用电影目标证据。
- 电视剧日历条目只有命中相同季号和明确集号范围时，才继承获取或入库证据。
- 只有“季包已完成”但没有内部文件集号时，该季各集保持未知。
- 'inLibrary' 只接受当前集明确的媒体库路径或同集入库证据。
- 季级 'chainId' 可以用于跳转任务详情，但季级健康状态不得直接覆盖每一集。

## 7. 日历接口兼容与体积

### 7.1 兼容参数

'GET /api/v2/calendar' 保留原有 'year'、'month' 和 'type' 行为，并新增可选参数：

- 'from=YYYY-MM-DD'；
- 'to=YYYY-MM-DD'；
- 'date=YYYY-MM-DD'；
- 'view=summary|detail'。

没有新参数时返回当前完整结构，旧客户端行为不变。

### 7.2 月摘要

'view=summary' 返回：

- 日期；
- 各状态数量；
- 当日前几条展示项；
- 月统计；
- 'version' 和 ETag；
- 是否存在更多详情。

月摘要目标体积小于 200KB。摘要不返回完整路径、阶段证据或全部作品详情。

### 7.3 当日详情

'date=YYYY-MM-DD&view=detail' 返回该日完整条目及集级证据。无效日期返回 '400 CALENDAR_RANGE_INVALID'。

新增字段全部可选，错误继续使用 v2 标准包络。

## 8. 移动端日历

### 8.1 导航

移动端固定五项：

- 首页；
- 发现；
- 追更；
- 任务中心；
- 日历。

### 8.2 月视图

- 使用七列紧凑日期格；
- 日期格只显示日期、状态点和数量；
- 不在格内塞入完整作品标题；
- 点击日期后按需请求当日详情；
- 不设置 860px 或其他桌面最小宽度；
- 页面不存在横向滚动。

### 8.3 周视图

移动端使用纵向日期列表，每日显示状态摘要和前几条作品。桌面端继续使用七列周视图。

## 9. 首页摘要

- 首页异常标题使用作品名和明确季集，例如《云月大陆》S01E05 识别失败。
- 原因使用用户语言，例如“Symedia 未找到对应媒体信息”。
- 路径、Hash、外部 ID、内部 Key 和原始错误只出现在任务技术详情。
- 正常保护不进入 'actionRequired'。
- 首页统计按唯一 'targetKey' 计算，串链冲突本身作为一项证据问题，不复制到全部候选目标。

## 10. RSS 身份

### 10.1 SQLite 字段

'rss_items' 增加：

- 'tmdb_id'；
- 'imdb_id'；
- 'identity_status'：'identified'、'conflict' 或 'unidentified'；
- 'identity_source'；
- 'identity_confidence'；
- 'identity_updated_at'。

迁移使用增量 'ALTER TABLE'，已有记录默认 'unidentified'。

### 10.2 身份来源

按以下顺序收集候选：

1. RSS/Atom 结构化扩展字段；
2. RSS 简介或公开链接中明确的 TMDB/IMDb ID；
3. 唯一可靠的现有追更匹配。

只有唯一候选可以写入 'identified'。多个不同 ID 写入 'conflict'，并保留候选来源；标题相似不能把追更身份反向写给种子。

本阶段不访问 PT 详情页。

### 10.3 中文详情抽屉

种子行可打开详情抽屉，展示：

- 中文字段名称；
- 来源和发布时间；
- 大小、分类、季集范围和版本摘要；
- TMDB/IMDb；
- 身份状态、来源和置信度；
- 已过滤的纯文本简介。

简介只按文本渲染，不使用 'dangerouslySetInnerHTML'。下载地址、完整详情地址和 Passkey 不返回浏览器。

'GET /api/v2/rss-items' 新增可选 'identityStatus' 筛选。旧请求和原有响应字段继续有效。

## 11. 桌面导航

桌面主导航最终保留：

- 首页；
- 发现；
- 追更；
- 任务中心；
- 日历。

影院大厅使用独立入口。种子库从主导航移除，在任务中心提供高级入口。旧 '/hall' 和 '/rss-library' 路由保持有效。

## 12. 实施与发布顺序

1. 增加证据所有权、匹配方式和冲突记录。
2. 只读运行新旧串链结果对比，不改变外部系统。
3. 修复无关电影继承同一 Symedia 文件的问题。
4. 上线结构化正常保护分类。
5. 建立集级证据集合。
6. 重新生成日历状态。
7. 增加日历摘要和日期详情参数。
8. 修改移动日历与五项导航。
9. 简化首页文案。
10. 接入 RSS 身份提取与中文详情抽屉。
11. 调整桌面导航和种子库入口。

每一步增加独立测试并保持可回滚。只读对比报告不得包含路径、Token、Cookie、Passkey 或外部鉴权参数。

## 13. 验收标准

1. 不相关电影不再继承《云月大陆》的错误。
2. 一条 qB/Symedia 证据最多只有一个目标所有者。
3. 串链结果不受输入遍历顺序影响。
4. 当前低分拒绝全部进入蓝色正常保护。
5. 没有集级证据的剧集不会显示已获取或已入库。
6. 'S00'、多集范围和明确季包文件能够形成正确集级证据。
7. 月日历使用摘要请求，不再一次返回完整详情，响应目标小于 200KB。
8. 手机导航可以直接进入日历。
9. 手机月视图和周视图都没有 860px 横向滚动。
10. 首页不显示路径、Hash 和内部 ID。
11. RSS 身份记录包含状态、来源、置信度和更新时间。
12. RSS 简介只作为纯文本显示。
13. 旧版日历和 RSS 请求继续正常返回。
14. 影视大厅现有视觉和播放流程不受影响。

## 14. 测试策略

- 单元测试覆盖空 TMDB、多候选、电影年份、电视剧季数和输入重排。
- 构造同一 Symedia 证据被 36 个电影候选命中的回归用例。
- 分类测试覆盖保护、真实失败、混合失败和未知证据。
- 集级解析测试覆盖单集、范围、多集、S00、绝对集数和季包文件。
- API 契约测试覆盖旧请求、新摘要、日期详情、非法范围和 ETag。
- RSS 测试覆盖结构化 ID、简介 ID、候选冲突、未识别和 HTML 过滤。
- 浏览器验收覆盖 1920×1080、1366×768、430×932 和 390×844 的深浅主题。
- 移动端对页面与日历容器执行横向溢出检查。
