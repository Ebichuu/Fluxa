# RSS 订阅资源精确搜索与海报回退设计

日期：2026-07-23  
状态：已确认，待实现

## 背景

追更详情中的“搜索资源”当前只把订阅标题和少量标题别名作为 RSS 全文查询条件。订阅已有 TMDB 身份、媒体类型和季号没有参与查询，因此中文标题与 RSS 中的英文名、原名或罗马音不一致时会得到 0 条。另一个问题是日常订阅列表只判断 `posterUrl` 是否存在，没有处理图片加载失败，失效的 TMDB/CDN 地址会直接显示浏览器裂图。

## 目标

- 让有可靠 TMDB 身份的订阅优先命中同一 TMDB 资源。
- 没有 RSS 身份时，只使用保守标题、媒体类型、年份和季号匹配，不扩大为模糊全量搜索。
- 任意非空但无法分词的查询返回空结果，绝不退化为最新种子列表。
- 订阅列表、发现卡片和订阅详情中的外部海报加载失败时显示标题首字占位，不出现裂图。
- 保持现有 RSS 查询、只读边界和敏感字段脱敏规则。

## 非目标

- 不访问 PT 详情页，不新增 Torra 下载或推送操作。
- 不根据相似标题反向写入 RSS 条目的 TMDB 身份。
- 不修改影院大厅的图片加载逻辑。

## 搜索设计

### 请求

前端调用现有 `GET /api/v2/rss-items`，继续保留 `query`、`sourceId`、`window`、`identityStatus`、`limit` 和 `offset`。增加可选目标字段：

- `tmdbId`：订阅的 TMDB ID；存在时作为首要筛选。
- `mediaType`：`tv` 或 `movie`。
- `seasonNumber`：电视剧目标季号。
- `year`：电影年份或电视剧首播年份。

旧请求不带这些字段时保持原有结构和状态码。

### 裁决顺序

1. `tmdbId` 与 RSS 条目的明确 TMDB ID 相同：精确命中。
2. RSS 条目有明确 TMDB ID 但与目标不同：排除。
3. RSS 条目没有 TMDB ID：必须同时满足媒体类型、可分词的规范标题/别名，以及电视剧季号或电影年份约束；无法形成唯一可靠标题条件时返回“证据不足”，不强行命中。
4. 查询文本非空但 `_match_query` 生成不了 token（例如片假名）：返回 0 条，并保留查询记录，不执行无条件 `SELECT`。

支持英文、数字、汉字、平假名和片假名的稳定化分词。前端最多提交四个标题候选，服务端按 RSS 条目 ID 去重；结果附带已有身份字段和可选 `matchMethod`/`matchConfidence`，供界面区分精确命中与标题兜底。

## 图片设计

- 新增统一的前端海报组件或等价小组件，接收 `src`、`title`、`className` 和尺寸状态。
- 对允许的外部图片地址优先使用现有 `/api/media/external-image?src=...` 安全代理，避免客户端直接依赖 TMDB/CDN 可达性。
- 代理或直链触发 `onError` 时只切换到首字占位，并记录一次本地调试事件；不得反复重试造成请求循环。
- 在追更列表和发现卡片接入该组件；日历现有 `EntryPoster` 行为保持不变但统一视觉占位。
- `posterUrl` 为空或加载失败都使用同一主题占位样式，浅色主题必须保证文字对比度。

## 错误与兼容

- RSS 查询参数非法返回现有 `422 RSS_QUERY_INVALID`。
- 目标筛选不会返回下载地址、详情地址、Cookie、Token 或 Passkey。
- 上游图片失败不影响订阅详情、搜索结果和页面交互；只显示占位。
- 没有匹配结果时文案明确为“没有找到与该订阅身份和季集相符的种子”，不再泛化为“本地种子箱为空”。

## 验收标准

1. `尼古喵喵 / ニャニャゴ` 不再因为片假名空查询返回最新 50 条无关种子。
2. 具有 TMDB ID 的订阅可命中同 ID RSS 条目，即使种子标题使用英文名或原名。
3. 无 TMDB 身份的电视剧必须匹配标题和季号；电影必须匹配标题和年份；多个候选时不绑定。
4. 无结果时响应总数为 0，且不会执行无条件列表查询。
5. 追更列表中的失效海报 URL 显示首字占位，不出现裂图或布局跳动。
6. 现有 RSS 列表、详情、身份筛选和旧客户端请求继续通过测试。
