# 全局浅色主题对比度与 RSS 回填提示设计

## 目标

修复管理工作台浅色主题下沿用深色主题浅色文字的问题，确保提示、状态、按钮、筛选器、表格和空状态在白色背景上保持可读；同时让 RSS“补齐身份”反馈明确说明本次运行结果。

## 范围

- 统一浅色主题的正文、次要文字、弱化文字、边框、面板和状态色变量。
- 为仍使用硬编码深色颜色的管理工作台组件增加浅色主题覆盖。
- 影视大厅保持现有沉浸式深色视觉，不参与全局浅色覆盖。
- RSS 回填成功/失败提示只修改文案和主题色，不改变回填行为或外部服务写操作。

## 设计规则

1. 浅色主题正文使用深灰，不使用白色或浅青色。
2. 次要文字与背景保持可读对比度；弱化文字只用于辅助信息，不用于操作结果。
3. 成功、保护、等待、异常、证据不足继续使用固定语义色，浅色主题使用更深的文字色和更淡的底色。
4. 所有按钮、筛选器和状态标签继承主题变量，不能直接继承深色主题的 `rgba(255,255,255,...)`。
5. RSS 回填提示展示：扫描数量、识别数量、冲突数量、未变化数量和剩余数量；失败时保留失败摘要。

## 验收

- 浅色主题下 RSS 回填提示、身份运行摘要、按钮、筛选器和状态标签清晰可读。
- 暗色主题视觉不回归。
- 影视大厅视觉不受影响。
- 前端 typecheck/build 通过。
- RSS 回填仍只写 Fluxa 本地身份和运行记录，不触发 Torra、qB、115、Symedia 或 Emby 写操作。
