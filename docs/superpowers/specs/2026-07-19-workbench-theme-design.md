# 非影院工作台白天 / 夜间主题设计

## 范围

本设计覆盖非影院工作台页面：总览、发现、订阅、订阅设置、种子库、任务中心、日历、控制室、设置及其弹窗。

明确排除：

- `src/components/media-hall/`
- 影院大厅专用样式、视觉控制台和视觉参数
- 后端 API、业务数据、RSS、Torra、qBittorrent、Symedia、Emby 和 MoviePilot 行为

## 已确认决策

- 主题只有白天和夜间两种状态。
- 用户选择写入 `localStorage`，刷新后保持不变。
- 首次访问优先使用系统 `prefers-color-scheme`，无法判断时默认夜间。
- 影院大厅隐藏主题按钮，并始终保持现有夜间视觉。
- 白天模式保留在线、警告、错误和完成等功能状态色。
- 采用 CSS 变量主题层，不新增一套独立的白天页面结构。

## 主题架构

### 状态所有权

`App` 统一管理 `ThemeMode = 'dark' | 'light'`：

1. 读取保存的 `mcc-ui-theme`。
2. 保存值不是 `dark` 或 `light` 时忽略。
3. 没有保存值时读取系统偏好。
4. 系统偏好不可用时使用 `dark`。
5. 切换后更新内存状态并写回 `localStorage`。

主题只在当前前端会话内生效，不请求后端，也不改变任何业务配置。

### 页面作用域

非影院页面的根节点使用 `data-theme="dark"` 或 `data-theme="light"`。影院大厅不挂主题属性，避免全局变量意外污染大厅。

白天覆盖规则统一使用：

```css
.app-shell:not(.app-shell--hall)[data-theme='light'] { /* theme tokens */ }
```

### 导航接口

`AppTopNav` 接收：

- `theme: ThemeMode`
- `showThemeToggle: boolean`
- `onToggleTheme: () => void`

主题按钮位于服务状态按钮和设置按钮之间。影院大厅传入 `showThemeToggle=false`，按钮不渲染。

按钮行为：

- 夜间状态显示太阳图标，标签为“切换到白天”。
- 白天状态显示月亮图标，标签为“切换到夜间”。
- 使用固定圆形触控尺寸、`aria-pressed`、`aria-label` 和 `title`。
- 不使用文字胶囊，避免右上角拥挤和移动端换行。

## 白天视觉令牌

白天模式采用 macOS 简白磨砂玻璃风格，但保持多个材质层级：

### 页面与材质

- 页面背景使用冷白和浅雾灰，移除夜间专用的彩色发光和大面积暗色渐变。
- 顶部导航使用半透明白色玻璃，较强模糊和轻微饱和度。
- 普通面板使用半透明白色、低对比度边框和柔和阴影。
- 重点面板使用更高不透明度和更清晰边缘，用于任务、服务状态和表单。
- 弹窗使用更厚的白色玻璃层、遮罩和更深阴影，维持顶层关系。
- 输入框使用浅灰底、明确焦点环，不使用纯白嵌套纯白。

### 令牌分组

白天令牌至少包含：

- 文字：主文字、次级文字、弱提示、交互文字
- 材质：`panel`、`panel-strong`、`console`、`glass-border`、`glass-shadow`、`focus-ring`
- 状态：在线、警告、错误、完成
- 交互：默认、悬停、按下、选中、禁用、聚焦

功能色继续用于状态识别，但会重新校准在浅色材质上的对比度。状态不能只依赖颜色，还要保留文字或图标。

白天主题不改变页面间距、卡片尺寸、导航位置、信息结构和顶部留白。

## 动效与交互

- 页面颜色、面板材质、边框和阴影在约 220–280ms 内平滑过渡。
- 不使用整页缩放、横向滑动或白屏闪烁。
- 主题按钮在按下时立即反馈，图标做轻微旋转与淡变，按钮尺寸保持固定。
- 切换可再次打断，动画从当前状态继续，不锁住输入。
- 不改变滚动位置、焦点位置、弹窗状态或当前页面。
- 不增加大面积背景循环动画。

## 无障碍与响应式

### 无障碍

- `prefers-reduced-motion: reduce`：取消位移和旋转，只保留短淡变。
- `prefers-reduced-transparency: reduce`：提高面板不透明度并关闭模糊。
- `prefers-contrast: more`：提高文字和边框对比度，使用更实的背景。
- 主题按钮支持键盘操作、可见焦点环、明确的 `aria-label` 和 `title`。
- 切换后提供可读的当前主题状态。

### 响应式

- 桌面端：按钮固定在服务状态和设置按钮之间。
- 平板端：保留图标按钮，不挤压主导航。
- 移动端：保留固定圆形触控区域，主导航继续横向滚动。
- 非影院页面切换不得造成横向溢出或布局跳动。
- 影院大厅隐藏按钮，保持现有布局和视觉控制台行为。

## 错误与边界处理

- `localStorage` 读取或写入失败时，主题仍可在当前会话切换。
- 保存值损坏时回退到系统偏好，再回退夜间。
- 主题切换不显示阻塞式弹窗，不依赖网络，不影响业务请求。
- 页面切换到影院大厅时，非影院主题状态保留；返回工作台后恢复原选择。

## 验收标准

1. 右上角主题按钮只在非影院页面出现。
2. 夜间切换到白天后，所有工作台页面、弹窗、表单和状态标签均使用白天材质。
3. 刷新页面后仍保持用户最后一次选择。
4. 首次访问遵循系统偏好，系统偏好不可用时为夜间。
5. 进入影院大厅后按钮隐藏，大厅视觉与现状一致。
6. 切换不改变页面位置、焦点、弹窗状态或业务数据。
7. 桌面和移动视口均无横向溢出。
8. 减少动效、减少透明度和高对比度设置下均有可用反馈。
9. 浏览器控制台无新增错误或警告。
10. `npm run build`、现有后端测试和 `git diff --check` 通过。
