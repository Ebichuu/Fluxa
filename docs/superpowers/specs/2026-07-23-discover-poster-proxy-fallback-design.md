# 发现页海报代理与失败占位修复

## 问题

发现页会把 TMDB、豆瓣、腾讯视频和爱奇艺图片转换为 `/api/image` 请求。该路由属于旧 Core Blueprint，当前被 `MCC_PRESERVED_CORE_API_ENABLED=false` 的总守卫误拦截，因此前端只能显示失败占位。放行测试同时确认：TMDB 返回有效图片，豆瓣部分地址会返回反爬 HTML，不能假定所有白名单地址都是真实图片。

现有失败占位使用铺满海报的大号首字。大量海报同时失败时会形成重复字块墙，不符合 Fluxa 工作台安静、清晰的视觉方向。

## 目标

- 恢复白名单来源的真实海报加载。
- 不开启任何其他旧 Core 管理接口。
- 不把 HTML、超大响应或非图片字节透传为图片。
- 图片确实不可用时，使用低干扰、深浅主题均清晰的稳定占位。
- 不改变发现页网格、卡片尺寸、订阅逻辑或来源数据。

## 方案

### 后端图片代理

1. 只将 `GET /api/image` 加入旧 Core 守卫的只读例外；其他旧接口继续返回 `PRESERVED_CORE_API_DISABLED`。
2. 继续使用现有固定图片域名白名单，拒绝任意主机、带凭据 URL 和无效协议。
3. 下载时保留来源所需的 Referer，并限制最大响应体。
4. 只接受 JPEG、PNG、GIF、WebP 和 AVIF 图片魔数；豆瓣反爬 HTML 等内容返回稳定错误，不进入浏览器图片缓存。
5. 成功图片增加公共缓存头，减少重复访问外部来源。

### 前端失败占位

发现卡片的失败占位改为中性底色、细边框和居中的 `ImageOff` 图标，移除铺满海报的大号首字。片名已经显示在海报下方，不在占位中重复。追更小海报和演员头像继续使用现有紧凑首字占位，避免小尺寸图标不可辨认。

浅色主题使用冷灰底和深灰图标，深色主题使用深灰底和低亮图标；不使用大面积蓝色、渐变或额外说明文字。

## 错误与安全边界

- 不支持的图片域名返回 400。
- 上游请求失败或返回非图片内容时返回 502，响应不包含外部响应正文、Cookie 或凭据。
- 图片代理是只读 GET，不写数据库、不触发订阅和下载。
- 失败后由 `PosterImage` 在当前卡片内切换占位，不产生破图图标或布局跳动。

## 验收

- `MCC_PRESERVED_CORE_API_ENABLED=false` 时，白名单 `/api/image` 请求仍可读取有效图片。
- 其他旧 Core 接口继续被 503 拦截。
- TMDB 样本返回图片字节和正确 Content-Type。
- 豆瓣反爬 HTML 不以 200 图片响应透传。
- 浅色和深色主题下失败占位均无大号首字、无白底白字。
- 发现卡片的 2:3 尺寸、网格列数和按钮位置不变化。
- 后端全量测试、前端类型检查和生产构建通过。
