# Fluxa 设置与工作台 UI 收口设计

日期：2026-07-20  
范围：除影视大厅外的工作台页面

## 目标

本轮集中解决三个已经在实机界面中暴露的问题：

1. 固定顶部导航在页面滚动后与任务内容视觉重叠。
2. 设置页大量配置使用英文变量名派生标题，普通用户难以理解用途。
3. 发现页右侧订阅详情在数据较少时存在大片空白，关键信息密度不足。

功能完整性优先，不删除现有可配置项，不改变影视大厅的独立视觉与交互。

## 方案选择

采用方案 A：滚动感知的完整磨砂顶栏、常用与高级配置分层、详情区按真实数据补齐。

未采用的方案：

- 始终显示不透明顶栏：能避免重叠，但首屏视觉过重。
- 将顶部导航改为普通文档流中的 `sticky` 元素：结构改动较大，容易影响所有页面既有间距。
- 删除旧兼容字段：违背“所有软件配置项均可在网页修改”的要求。

## 一、顶部导航防遮挡

### 当前问题

`AppTopNav` 使用固定定位，页面内容在滚动时会进入导航区域。导航目前只有按钮和胶囊自身拥有磨砂背景，胶囊之间仍然透明，因此任务统计、标签等文字会透出来，看起来像两层内容重叠。

### 设计

- 保持导航固定定位和现有位置，避免改变页面首屏布局。
- 在非影视大厅页面监听窗口滚动状态。
- 页面位于顶部时保持当前轻量悬浮效果。
- 滚动超过小阈值后，为整个导航安全区显示连续的半透明磨砂底板、柔和阴影和一条细分隔线。
- 顶栏底板覆盖视口顶部到导航控件下方，不留下可透出文字的空隙。
- 深色与浅色主题分别使用对应材质变量；浅色保持 macOS 简白磨砂感。
- `prefers-reduced-transparency` 下使用不透明背景，`prefers-reduced-motion` 下取消渐变动画。
- 影视大厅不使用该滚动底板。

### 验收

- 1920×1080 下滚动任务中心，任务摘要、卡片和工具栏不再透过导航显示。
- 发现、订阅、种子库、控制室、日历、设置页具有一致行为。
- 回到页面顶部后底板自然淡出，无闪烁和布局跳动。

## 二、设置项中文化与分层

### 原则

- 所有当前允许网页修改的配置继续保留。
- 常用设置优先展示；旧 NasEmby 自动化和兼容字段进入“高级兼容设置”，默认折叠。
- 用户界面以中文名称和用途说明为主，环境变量名不再作为主要标题。
- 提供“显示技术字段名”开关，排查问题时仍可查看 `ENV_*`、`MCC_*` 等真实键名。
- 密码、Token、Cookie 等敏感值继续采用“已保存时不回显，重新输入才修改”的行为。

### 字段元数据

为设置字段补充显式元数据：

- 中文名称。
- 一句话用途说明。
- 所属分组。
- 是否高级兼容字段。
- 是否敏感。
- 是否需要重启。
- 依赖条件或“未使用该功能可留空”。

重点覆盖以下字段族：

- `MCC_*` 与 `NASEMBY_*` 自动化开关。
- `ENV_SYMEDIA_*` 旧 Symedia 自动订阅兼容配置。
- 115 上传目录、清理目录和旧 TGMonitor 配置。
- `ENV_PTTO115_*`、`ENV_PTTO123_*` 旧转存流程。
- Telegram 账号登录、Bot 通知和订阅通知配置。
- 旧媒体库账号别名与其他 `ENV_*` 兼容字段。

自动化开关使用明确中文，例如“启用私人 RSS 采集”“允许订阅写入”“启用 Torra 质量观察”，并说明关闭时影响，避免只显示英文变量名与开关状态。

### 页面结构

- “常用配置”显示当前主链真正使用的连接、订阅、RSS、下载器、媒体库和通知设置。
- “高级兼容设置”默认折叠，并在标题处说明通常无需修改。
- 技术变量名默认隐藏；开启开关后显示在中文名称下方。
- 搜索同时匹配中文名称、说明和技术变量名，隐藏技术名不影响搜索。

### 验收

- 默认界面不再出现 `Env Symedia Token`、`Mcc Private Rss Enabled` 这类机器翻译标题。
- 用户能从中文说明判断配置用途、依赖和是否可以留空。
- 所有现有配置仍能保存，敏感值不会在读取接口或页面中回显。

## 三、发现页订阅详情填充

### 原则

只展示真实可用信息，不增加纯装饰占位。

### 信息层级

右侧订阅详情按以下顺序组织：

1. 媒体摘要：类型、年份、语言、TMDB 信息。
2. 订阅范围：电影或剧集季数、已选季、分集与缺集进度。
3. 当前状态：是否已订阅、Torra 镜像状态、入库路径和 Emby 入库状态。
4. 获取策略：默认 PT/Torra、质量观察状态、MoviePilot 备用状态。
5. 可执行操作：加入订阅、立即运行、打开对应订阅详情或任务中心。

当某类数据缺失时显示简短的可理解状态，不保留固定的大块空白。左右列不再通过无意义的最小高度强制等高；右侧内容保持紧凑，必要时使用底部操作区承接剩余空间。

### 验收

- 1920×1080 下选择电影和剧集各一项，右侧详情均无突兀空白。
- 季集、入库、Torra 和备用通道信息仅在有意义时展示。
- 加入订阅和跳转操作始终清晰可见。

## 实施边界

- 不修改影视大厅页面及 Mineradio 来源代码。
- 不改变配置 API 的键名和已有保存格式。
- 不在本轮加入 Torra 远端删除；第一阶段仍保持单向镜像和只读保护。
- 不启用用户未开启的自动化能力。

## 验证计划

- 前端类型检查、单元测试和生产构建。
- 后端运行时设置相关测试，验证分组、字段元数据、保存和敏感值保护。
- 1920×1080 深色与浅色主题检查。
- 滚动检查所有非影视大厅页面的顶部安全区。
- 设置页检查常用/高级分层、技术字段开关和搜索。
- 发现页检查电影、剧集、已有订阅和未订阅四种详情状态。
