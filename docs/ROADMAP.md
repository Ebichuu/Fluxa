## 2. 当前下一阶段：候选镜像与 fnOS 实机

1. M-Team、HDHome、织梦、青蛙四个真实结构脱敏夹具已满足当前版本；后续新增站点样本只扩展兼容性，不阻塞部署。
2. 四个来源的 RSS 2.0 字段、电影/单集/整季包、多版本、文件大小、enclosure 和 `720p/1080i/1080p/2160p` 已由脱敏夹具复核；`429/Retry-After`、指数退避、并发 2 和抓取记录上限已经完成。
3. 临时 SQLite 原子替换式迁移演练已经完成；本次 fnOS 没有旧生产数据，首次部署直接初始化空库，Docker 重启持久化已经通过。
4. Torra 幂等、冷却、外部 job ID 和按集追更洗版状态已持久化；洗版分析/下载/job 适配器与统一动作查询已完成。
5. Emby 基准、观察单元、RSS 条目/季集本地匹配、一次性 Torra 分析唤醒、默认关闭的有限主动调度、阶段 6 人工 API 和 React 页面已完成；只剩实机验证。
6. 当前版本进入 Emby 成为追更洗版基准后，每条订阅选择 24 或 48 小时窗口；RSS 新条目即时唤醒。主动搜索默认使用 `12 / 24` 或 `12 / 24 / 48` 小时兜底，也可配置最早 30 分钟的自定义时间点。
7. 当前集到期后停止自动搜索，版本升级不延长窗口；下一集入库后重新建立窗口，不补扫历史订阅。
8. 版本优先级沿用 Torra 规则；中控只选择 Torra 返回的正分差最高候选，并负责限频、阻止重复和展示状态。
9. 本地模拟测试已完成；构建新候选镜像后再进入 fnOS。

剩余实施顺序以本路线图、`docs/CORE_API_CAPABILITY_MATRIX.md` 和 `docs/DEPLOYMENT.md` 为准。

## 3. 后续：PT 主链实机闭环

主线固定：

```text
媒体控制中心订阅
  → Torra
  → qBittorrent
  → Torra 秒传 115
  → Symedia
  → Emby
```

实施顺序：

1. fnOS 单容器只读部署。
2. 核对 Torra 下载器 ID、八分类保存路径和 115 秒传目标。
3. 创建一条电影测试订阅，只读取 Torra 推送预览。
4. 用户单独批准后开启一次 Torra 推送。
5. 观察 qB 下载、Torra 秒传、Symedia 接管和 Emby 索引证据。
6. 电影闭环成立后验证一条剧集季。
7. 最后开启订阅调度。

当前不执行以上实机动作。

## 4. MoviePilot 其他 PT 站点备用

MoviePilot 不是 Telegram 网盘通道。后续目标：

- Torra 追更洗版和版本控制永远优先。
- MoviePilot 自身数据库稳定性不作为中控可修复问题，因此默认关闭。
- 只有用户明确开启、Torra 不覆盖目标站点或观察已经结束，且没有 Torra/qB 活动任务时，才允许 MoviePilot 参与。
- MoviePilot 下载继续进入同一 qB → 115 → Symedia → Emby 主线。
- 不允许 Torra 与 MoviePilot 对同一媒体并行下载。
- 使用现有 `moviepilot_status()`、`moviepilot_subscribe()`，并以 [MoviePilot 官方文档](https://wiki.movie-pilot.org/) 核对接口，不猜测。
- 本轮已补齐安全预览和人工备用入口；MoviePilot 自动模式不实施，真实动作仍需受控实机窗口。

## 5. Telegram 网盘能力（延期但保留）

未来范围仍包括：

- Telegram 登录、频道选择和频道资源订阅。
- HDHive / pansou 授权、搜索、签到和影巢解锁。
- 115 分享链接候选预览与单条转存。
- PT 失败后的人工或自动网盘兜底。
- 真实 115 转存直接证据、失败对账和防重复。
- 115 清理和助力；这两项风险较高，继续排在最后。

保留边界：

- Python 模块、原路由、v2 安全适配层和模拟测试不得删除。
- 当前 React 不引用这些动作接口。
- 所有环境闸门保持关闭。
- 不启动 NasEmby `legacy/ptto115.py`；PT 主线秒传由 Torra 独占。
- 未来重新启用前必须另写设计并重新核对 Torra、qB、115、Symedia 和 Emby 重复证据。

重新启用前应基于当前接口和安全边界单独设计，不沿用已归档的迁移步骤。

## 6. 其他未完成工作

### 任务链证据

- 获取 Torra 秒传到 115 的直接结果；接口没有该证据时继续明确标记推断。
- 无法可靠关联的外部记录继续显示未关联，不猜测合并。
- 实机确认八分类在 qB 保存路径、115 待整理目录和 Symedia 归档之间一致。

### 受控动作

- qB 暂停/恢复真实任务验证。
- Emby 证据刷新真实验证。
- Torra 单条真实推送与冷却验证。

### 部署

- fnOS 单容器只读部署。
- HTTPS 反向代理和源站端口限制。
- 持久目录备份、升级和回滚演练。
- 多 worker 或多副本前增加调度选主和台账并发控制。

## 7. 明确不做

- 不建立网页播放器；播放交给 Emby 客户端。
- 不建立 PT 站点爬虫或第二套质量评分器；PT 搜索和版本高低判断交给 Torra，MoviePilot 只作可关闭备用。
- 不让 NasEmby `ptto115.py` 与 Torra 秒传同时运行。
- 不向 Symedia 重复推送订阅；Symedia 只负责 115 后整理入库。
- 不把 NasEmby 原静态管理页面作为第二套用户界面启用。
- 不建立第二套订阅台账或第二套调度器。
