# 前端设计说明

## 模块职责

`src/` 是 Fluxa 的 React 工作台前端，负责发现、订阅、种子库、任务、活动和设置等统一操作界面。影院大厅使用 Mineradio 提供的独立视觉资源和页面边界，不继承工作台的主题与布局规则。

页面通过 `src/services/api.ts` 访问同源 Python API。前端只负责展示状态和发起明确操作，不持有外部服务密码、Token、RSS 下载地址或写权限判断。

## 信息架构

- `components/pages/`：按用户任务组织页面和详情交互。
- `components/layout/`：导航、确认弹窗和跨页面框架。
- `services/api.ts`：统一 HTTP 请求、超时与错误处理。
- `types/`：前后端公开响应类型。
- `styles/`：工作台、发现页和影院大厅分层样式。

工作台以宽屏电脑使用为主，同时保留窄屏可操作性。深色和浅色主题共享信息层级；浅色使用克制的 macOS 磨砂玻璃风格。弹窗和浮层必须支持键盘焦点、关闭恢复和减少动态效果。

## PT 获取主线

```text
发现内容 -> 本地订阅台账 -> Torra -> qBittorrent -> Torra 秒传 115 -> Symedia -> Emby
```

Torra 是第一阶段唯一 PT 获取入口。发现页和订阅详情中的“搜索资源”只调用 `GET /api/v2/rss-items` 查询本地 SQLite RSS 种子箱，用于查看候选证据，不触发下载、转存、订阅推送或其他外部写操作。

每个 RSS 来源可独立设置轮询周期。界面保留 1、3、5 分钟快捷值，同时允许输入 1–1440 分钟的自定义整数；后端执行相同范围校验。

订阅搜索会依次使用本地标题、TMDB 中文标题、英文标题和原始标题查询，再按种子 ID 合并去重。查询中、失败、无结果和结果列表直接显示在被点击的订阅条目内；发现页的同类结果插入被点击卡片所在网格位置，避免用户离开当前上下文。

订阅详情进入任务中心时携带订阅 ID、TMDB ID、季号和标题。任务中心按订阅 ID、TMDB ID + 季号、规范化标题逐级匹配；命中后显示该订阅的全部关联任务并聚焦第一条，未命中时明确说明订阅尚未形成任务。用户可随时退出聚焦并恢复普通筛选列表。

旧的 `/api/discover/resources/search` 作为后端兼容接口保留，但当前 React 工作台不调用它。115、Telegram、HDHive 和 MoviePilot 不属于当前资源搜索路径。

## 状态与安全边界

- 敏感字段仅显示“已保存”状态，前端不回显原值；只有重新输入才修改。
- 所有外部写操作由后端配置闸门最终判断，前端按钮状态不能代替服务端授权。
- `TORRA_PUSH_ENABLED=false` 时，保存订阅只进入本地台账，不应产生 Torra 后台排队或失败活动。
- Torra 第一阶段采用单向只读镜像；远端删除和双向冲突处理留到第二阶段。
- RSS 搜索结果不暴露下载 URL、Passkey 或 Cookie，只展示来源、标题、时间、大小、版本和季集信息。

## 变更记录

### 2026-07-22 - 首页结论与追更三维状态

首页改为只读取统一状态摘要，不再由浏览器根据 Torra、qB、Symedia 和 Emby 接口是否返回成功拼装绿色结论。追更工作台同时展示对账状态、履约状态和健康状态，并在本地台账为空时只读展示 Torra 现有订阅。标题相同只能产生身份冲突，不能自动判为已关联；发现页新增作品明确提示“已保存追更意图，尚未同步到 Torra”。影院大厅未修改。

### 2026-07-22 - 任务链身份 v2

任务中心开始读取 `/api/v2/tasks/chains`，使用稳定 `chainId` 串联 `mediaKey`、`targetKey` 和具体 qB/远端文件 `artifactKeys`；旧任务字段保持兼容。阶段证据返回观察时间、来源、原因和可用动作资格，暂不开放新的外部写操作。

### 2026-07-21 - PT 优先与本地 RSS 搜索

发现页和订阅详情的资源入口改为本地 RSS 种子箱；Torra 推送开关关闭时在后台队列创建前停止处理。此变更收紧了搜索和外部写入边界，不改变国产平台榜单、影院大厅或手动 Torra 推送确认流程。

### 2026-07-21 - 订阅搜索反馈与任务定位

订阅资源搜索增加 TMDB 多标题查询、结果合并和条目内即时反馈；任务中心增加订阅上下文定位、分级匹配、目标任务聚焦及独立无任务状态。影院大厅、任务链数据结构和外部写入权限保持不变。

### 2026-07-21 - 工作台按钮行为收口

工作台按钮统一遵循四项规则：异步刷新、测试和保存动作必须有进行中状态并防止重复提交；保存期间锁定相关输入，避免响应覆盖用户的新修改；跨页面跳转必须携带订阅、TMDB、季号或标题等目标上下文；尚未实现的永久禁用按钮不保留为可见伪操作。日历逾期条目现在可直接定位到对应任务，控制室、任务中心、种子库和设置页的失败或进行中状态会在当前页面反馈。

### 2026-07-21 - 第一阶段订阅工作台闭环

订阅页改为读取统一的工作台快照，集中展示本地写入、Torra 连接、镜像同步、RSS 和定时任务五项真实能力，以及电影、剧集、待处理和已入库统计。空台账按“预览 Torra 订阅、检查冲突、确认导入”引导；列表直接显示订阅范围、缺集、Torra、qB、115、入库状态和阻塞原因。自动订阅来源刷新采用增量合并，保留手动订阅和 Torra 镜像；第一阶段仍不提供 Torra 远端删除或反向编辑。
